Optionally seed the OpenClaw workspace on first boot from a Git repository or an uploaded folder. Selecting a source switches the instance to user-managed config.

What to provide

A directory tree that seeds the Claw's OpenClaw home (~/.openclaw, on its persistent volume). It is applied once on first boot; edits made later inside the running Claw are preserved.

  • workspace-main/ → the agent workspace (~/.openclaw/workspace/): e.g. AGENTS.md, SOUL.md, memory/, skills/
  • openclaw.json → merged into the Claw's effective config (not copied as a workspace file)
  • any other top-level folder → copied under ~/.openclaw/ (e.g. skills/ → ~/.openclaw/skills/)

For Git, the repository root is used, or the subpath you set below.

Full layout reference and examples: redhat-et/claw-collections.
